Learning Objective 01

Probability as a foundation of statistical modeling, including inference and maximum likelihood estimation

Statistical modeling is not just curve-fitting — it is the act of making probability statements about the world given data. This objective asks whether I understand that foundation: what a p-value means, what a confidence interval commits us to, and how maximum likelihood connects a model's distributional assumptions to the estimates we compute.

Show: Homework I — Inference from Simple Linear Regression

Evidence · Homework-I.Rmd / Homework-I.pdf

In Question 1 (Auto dataset, mpg ~ horsepower), I used TidyModels to fit a simple linear regression and then carefully distinguished between a confidence interval and a prediction interval for horsepower = 98.

# 95% confidence interval for the MEAN mpg at horsepower=98 conf_int <- predict(model_auto, new_data=test_auto_data, type="conf_int") # (.pred_lower=24.0, .pred_upper=25.0) # 95% prediction interval for a SINGLE car at horsepower=98 pred_int <- predict(model_auto, new_data=test_auto_data, type="pred_int") # (.pred_lower=14.8, .pred_upper=34.1)

The distinction matters deeply. The confidence interval is narrow (24.0–25.0) because it captures our uncertainty about the average response — averaging over many cars reduces noise. The prediction interval is wide (14.8–34.1) because individual cars vary around that average. This is a direct consequence of how OLS partitions uncertainty: the confidence interval reflects only estimation error (the sampling distribution of β̂), while the prediction interval adds residual variance σ². I wrote this out explicitly in the homework as a probability statement — "We are 95% confident that the average mpg of cars with horsepower 98 falls between 23.97 and 24.96" — which is a commitment about a probability model, not just a number.

Tell: Why this demonstrates the objective

Maximum likelihood estimation underlies OLS when we assume normally distributed errors: the OLS estimates are exactly the MLEs of β under that model. Every p-value and confidence interval I computed in HW-I flows from this assumption — the t-statistics follow a t-distribution because of the normality assumption on ε. When I checked the Q-Q plot and found non-normality (the residuals deviated in the tails), I was effectively checking whether the probability model justifying our inference held. I concluded it did not, and said so explicitly. That willingness to reject the probabilistic foundation when the evidence demands it is, I think, the clearest evidence that I understand what probability is doing here — it is not decoration, it is the engine of inference.

Supporting evidence · Homework-I, Q2

In the bias-variance question (Q2), I explained why a cubic model has lower training RSS but higher expected test RSS when the true relationship is linear — reasoning entirely in terms of the bias-variance tradeoff, which is itself a statement about expectations under a probability model.

Supporting: Project 1 — Coefficient Significance as Probabilistic Claims

In the diamonds regression, all five coefficients of our final polynomial model were significant at p < 0.001. Reporting significance is easy; understanding it means grasping that under H₀ (a coefficient is zero), the t-statistic follows a known distribution, and our observed value was so extreme we would expect it less than 0.1% of the time by chance alone. The large sample size (34,000+ training observations) also illustrated the central limit theorem in action — even mild non-normality in the residuals becomes tolerable because the sampling distribution of β̂ is approximately normal.

Learning Objective 02

Apply the appropriate generalized linear model for a specific data context

Choosing the right model is not about picking the most sophisticated option — it is about matching the model's assumptions to what the data actually are. This objective is best demonstrated through Project 2, where we faced three distinct outcome types and had to make principled choices for each.

Show: Project 2 — Four Models, Four Justified Choices

Evidence · project2_classification_k12.Rmd / project2_classification_k12.docx

Using the 2019 Parent and Family Involvement (PFI) Survey (15,374 K–12 students), we modeled two distinct outcomes: (1) how often parents participate in school activities (FSFREQ — a count), and (2) a child's grade category (Mostly A's / B's / C's / D's or below).

The Poisson → Negative Binomial Decision

FSFREQ is a non-negative integer count (0–99), which makes Poisson regression the natural starting point. But we did not stop there. Before interpreting any Poisson results, we checked the core assumption: that the variance equals the mean.

Mean (FSFREQ)7.02
Variance (FSFREQ)85.4
Variance/Mean ratio12.2×
Poisson deviance ratio5.12

A ratio of 12.2 — far above the value of 1 that Poisson requires — is severe overdispersion. Fitting Poisson anyway would produce standard errors that are too small, making predictors look significant when they are not. We therefore fit a Negative Binomial model, which adds a dispersion parameter θ allowing variance to exceed the mean. A likelihood ratio test confirmed the NB model was significantly better (LR = 60.3, p < 0.001), and the AIC improved by approximately 35,000 points. This is not a small tweak — it is the difference between valid and invalid inference.

# Overdispersion: Poisson residual deviance / df >> 1 # Deviance = 62,904 | df = 12,281 | ratio = 5.12 → invalid # Negative Binomial: adds dispersion parameter θ nb_spec <- poisson_reg() |> set_engine("glm.nb") |> set_mode("regression")

Logistic Regression for Binary High-Achiever Outcome

For the binary outcome (high_achiever: Mostly A's vs. not), logistic regression was appropriate — it models log-odds as a linear combination of predictors, producing interpretable odds ratios. We confirmed no multicollinearity (all GVIF values < 2) and validated on a held-out test set, achieving AUC = 0.743. Ten-fold cross-validation (CV AUC = 0.726) confirmed the result was not an artifact of the split.

Multinomial Logistic Regression for Four Grade Categories

When we expanded the outcome to all four grade categories, binary logistic regression was no longer appropriate — collapsing four unordered categories to two would discard information. Multinomial logistic regression fits K−1 = 3 simultaneous equations, each comparing one grade category to the baseline (Mostly A's). We also addressed class imbalance (6,051 A students vs. 194 D students, a 31:1 ratio) using step_upsample(), applied only to training data to avoid leaking test-set information.

LDA as a Theoretically Distinct Approach

We also applied Linear Discriminant Analysis (LDA) and Quadratic Discriminant Analysis (QDA) as a theoretically distinct classification approach. Where logistic regression models P(Y=k|X) directly, LDA models P(X|Y=k) and uses Bayes' theorem. Using five continuous predictors (school involvement score, home engagement score, income, family dinner frequency, and disability as a continuous proxy), LDA achieved AUC = 0.651 — substantially below logistic regression's 0.743. This gap was principled and expected: LDA could not use the categorical predictors (race, parent education, disability status) that drove logistic regression's performance.

Key insight: Choosing the right GLM means not just picking from a menu, but diagnosing when your first choice fails its own assumptions (Poisson → NB) and understanding why two valid methods can produce different results by design (logistic vs. LDA).

Supporting: Homework IV — Multinomial Logistic Regression (Nonvoters)

Evidence · Homework-IV.Rmd / Homework-IV.pdf

Using the FiveThirtyEight nonvoters survey (5,836 respondents), I fit multinomial logistic regression for a three-level outcome: voter_category ∈ {rarely/never, sporadic, always}. I extended the class model by adding party identification as a predictor, compared models with and without it, and interpreted both log-odds and odds-ratio form coefficients.

Democrats had 4.06× higher odds of being an "always" voter vs. "rarely/never" compared to those identifying as "Other" (β = 1.401, the exponentiated coefficient e^1.401 ≈ 4.06), while Republicans showed 3.45× higher odds (β = 1.239). Adding party reduced the gender coefficient slightly (from −0.211 to −0.192), suggesting that some of the gender–voting relationship is mediated by party identity.

Learning Objective 03

Demonstrate model selection given a set of candidate models

Model selection is the process of choosing from a set of candidates in a way that is honest — that does not overfit to data we happen to have, and does not use the same data to both select and evaluate. I have demonstrated this in two very different ways: through exhaustive best-subset selection in Project 1, and through stepwise backward elimination in HW-III.

Show: Project 1 — Best-Subset Selection with Held-Out Validation

Evidence · project1_diamonds_linear_regression.Rmd

With 25 candidate predictors (raw dimensions x, y, z; their squares; table; depth; and dummy expansions of cut, color, clarity), we used exhaustive best-subset selection via regsubsets() to identify the best-fitting model of each size k = 1, …, 25.

The critical methodological decision was to not use training-set criteria alone. We implemented a three-way split:

# Three-way holdout — data leakage prevention set.seed(42) diamonds_split <- initial_split(diamonds_clean, prop = 0.80) train_val <- training(diamonds_split) # 80% test_set <- testing(diamonds_split) # 20% — NEVER touched until end val_split <- initial_split(train_val, prop = 0.80) train_set <- training(val_split) # 64% — model fitting val_set <- testing(val_split) # 16% — model selection

We computed validation-set MSE for each of the 25 best models. It was minimized at k = 5 predictors: table, x, z, x², z². Every quality-grade variable (cut, color, clarity) was excluded at this optimal size — a finding that is substantively interesting (once dimensions are known, grade adds nothing to carat prediction) and methodologically trustworthy because it came from data the model never saw during fitting.

Best model size5 predictors
Test R²0.997
Test RMSE0.026 carats
ExcludedCut, Color, Clarity
Why the spike matters: Validation MSE jumped sharply when moving from 7 to 8 predictors — adding the first categorical variable triggered multicollinearity (x, y, z are highly correlated) and destabilized coefficients. This is exactly the kind of insight that in-sample criteria like AIC/BIC can miss, and it required held-out validation to detect.

Supporting: Project 2 — Likelihood Ratio Tests for Model Comparison

In Project 2, model selection used likelihood ratio tests rather than information criteria or validation MSE. For both the Negative Binomial model (full vs. reduced: LR = 60.3, p < 0.001) and the logistic model (full vs. reduced: deviance reduction = 177.5, p < 0.001), the full model was significantly preferred. AIC comparisons corroborated these decisions. This is a formally different selection strategy from best-subset or backward elimination, and demonstrates that I can apply the appropriate selection tool for the modeling context at hand.

Learning Objective 04

Express the results of statistical models to a general audience

Numbers from a model mean nothing until they are explained to someone who did not fit the model. This objective is about translating statistical outputs — coefficients, odds ratios, confidence intervals — into language that a non-statistician can act on. It was one of my explicit goals at the start of the semester, and I can trace my development through the work.

Show: Project 1 Report — The Jeweler's Insight

Evidence · project1_diamonds_linear_regression_report.pdf

The report opens with a research question a general audience can immediately grasp — can a jeweler estimate carat weight without a scale? — and ends with a practical implication that answers it directly.

The key communication move was translating a test R² of 0.997 and RMSE of 0.026 carats into something tangible: "A jeweler equipped with a millimeter gauge and a table-angle tool — both non-contact instruments — could estimate any diamond's carat weight to within roughly 0.026 carats, without a scale." This is the same number, but expressed in terms of what it means for a person who works with diamonds.

We also made the counterintuitive finding legible: quality grades (cut, color, clarity) — the features most buyers focus on — add no predictive power once physical dimensions are known. We explained why: large diamonds are hard to cut and clarity-grade perfectly, so high-quality grades tend to appear on smaller stones. This transforms a regression coefficient exclusion into a market insight.

Show: Homework II — Coefficient Interpretations with Interaction Terms

Evidence · Homework-II.Rmd / Homework-II.pdf

Interpreting a model with interaction terms (Price × ShelveLoc) requires care, because the "effect" of price on sales depends on which shelf location you are in. I wrote out three explicit sub-equations and then translated them into plain language.

For example, for the interaction coefficient β₇ (Price × ShelveLocMedium = +0.0063):

Plain-language interpretation (Homework-II)

"Keeping everything else constant, when carseats are placed on a medium quality shelf location as opposed to a bad shelf location, the sales volume decreases by (62 − 6) = 56 units for every unit increase in price."

This kind of "net effect" interpretation — combining main effects and interaction terms into a single actionable sentence — is what communication to a general audience requires. A retailer does not care about β₁ and β₇ individually; they want to know: if I raise the price by $1 on a medium shelf, what happens to sales?

Show: Homework IV — Odds Ratio Communication

Odds ratios are famously hard to communicate. In HW-IV, I translated the party coefficients into percentage-change language:

Plain-language interpretation (Homework-IV)

"The odds of being an 'always' voter (vs. 'rarely/never') increases by a factor of 4.06 (~406%) for a Democrat compared to someone who identifies with no major party, holding all other variables constant."

I also used the model comparison (with/without party) to tell a causal story: adding party reduced the gender coefficient from −0.211 to −0.192, suggesting "some of the association between gender and voting may be explained by party" — a mediation framing that is accessible without invoking formal mediation analysis.

Learning Objective 05

Use programming software to fit and assess statistical models

TidyModels provides a unified interface for fitting, assessing, and comparing models across engines and families. This objective is about fluency with that interface — not just running code, but understanding the workflow and making deliberate implementation choices.

Show: Homework II — The Full TidyModels Pipeline

Evidence · Homework-II.Rmd / Homework-II.pdf

The Carseats analysis walks through the complete TidyModels workflow: specification → data splitting → fitting → coefficient extraction → model assessment → prediction on new data → diagnostic visualization.

# 1. Specification lm_spec <- linear_reg() |> set_engine("lm") |> set_mode("regression") # 2. Train/test split carseats_split <- initial_split(carseats, prop = 0.80) carseats_train <- training(carseats_split) carseats_test <- testing(carseats_split) # 3. Fit with interaction model <- lm_spec |> fit(Sales ~ Price * ShelveLoc + Age + Advertising, data = carseats_train) # 4. Assess — tidy() / glance() / augment() tidy(model) # coefficients with SEs and p-values glance(model) # R², adj. R², AIC, BIC augment(model, new_data = carseats_test) # predictions + residuals # 5. Test set evaluation carseats_test_aug |> rsq(truth = Sales, estimate = .pred) # Test R² = 0.726 vs Train R² = 0.675 → generalizes well

Show: Homework III — Reusable Functions as Software Craft

Evidence · Homework-III.Rmd

Rather than copy-pasting the same assessment code for each backward-elimination step, I wrote a reusable report_step() function that encapsulates the repetitive pattern.

report_step <- function(model, step_title="Step 1", alpha = 0.05) { td <- tidy(model) gl <- glance(model) # candidates for removal: p.value > alpha (exclude intercept) candidates <- td |> filter(term != "(Intercept)", p.value > alpha) |> arrange(desc(p.value)) print(step_title) print(td) print(gl) print(candidates) }

This is a small thing, but it reflects a programmer's mindset: identifying repetition, abstracting it, and making the code readable. It also made the backward-elimination output consistent across all four steps, which made comparing adjusted R² values straightforward.

Show: Project 2 — Advanced TidyModels Features

Project 2 pushed our software use into more advanced territory:

  • step_upsample(over_ratio = 0.5) from the {themis} package — applied only within the training fold to prevent leakage
  • repair_call() — a TidyModels utility to fix an internal reference bug in multinomial models so that augment() works correctly
  • 10-fold cross-validation with vfold_cv() and fit_resamples() — to confirm test-set performance was not a lucky split
  • Workflow objects (workflow()) combining recipe steps and model specifications — keeping preprocessing and modeling together and preventing data leakage

Show: Homework IV — ggbivariate and Functional Abstraction

In HW-IV, I re-implemented the ggbivariate() helper from the class Rmd rather than copying it blindly — adapting it to return a list of ggplot objects when multiple predictors are passed, which is more flexible. This required understanding ggplot's aes(x = .data[[p]]) pattern for tidy evaluation inside a function. Small moments of genuine software problem-solving like this accumulate into fluency.

Reflection on software growth: My background as a machine learning engineer meant I was comfortable with Python and sklearn before this course. Learning TidyModels taught me a different philosophy — one where each step (recipe, model spec, workflow, resampling) is explicit and auditable. I now find myself applying this kind of pipeline thinking back in my ML work.
